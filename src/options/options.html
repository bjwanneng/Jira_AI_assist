Google Drive

Google Drive uses OAuth 2.0, not a static token. You need to set up a Google Cloud OAuth Client ID first:

  1. Go to console.cloud.google.com → create or select a project.
  2. APIs & Services → OAuth consent screen → External → add yourself as a test user.
  3. APIs & Services → Library → enable Google Drive API, Google Docs API, Google Sheets API.
  4. APIs & Services → Credentials → Create Credentials → OAuth Client ID → Application type: Chrome Extension.
  5. Item ID: ... (your extension ID — also find it on chrome://extensions). Replace if you load from a different folder.
  6. Copy the Client ID (ends in .apps.googleusercontent.com) and paste it into manifest.json → oauth2.client_id.
  7. Reload the extension.

SiFive Document Library

Upload SiFive product user guides and manuals (PDFs, 250-600 pages). Documents are parsed and indexed locally in your browser; no PDF content is uploaded to any cloud service except the LLM endpoint you configured for embeddings.

Each document is tagged by product (U74, E76, ...), config (W3K, FA7, ...), and customer (general or company name). The assistant auto-loads relevant docs based on the conversation context.
